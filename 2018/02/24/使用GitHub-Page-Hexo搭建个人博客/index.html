使用GitHub Page+Hexo搭建个人博客

发表于 2018-02-24 | 分类于 hexo
记录一下github page的使用和一些踩过的坑

前言

  GitHub是个好东西，除了可以很快的查询到一些有用的代码(笑)，还可以分享代码，解决技术难题等多种功能。GitHub Page是GitHub提供给开发人员在其之上创建静态网站的一个挺有意思的功能，方便我们这一类懒得去打理服务器的懒癌患者。当然，只能创建静态网站，总不能和云机大佬抢饭吃嘛…
  这里整理下我自己的整个部署的过程和踩过的坑，给自己一个日后的回忆，如果顺便帮助到有需要的人就最好啦(笑)

环境和一些准备

  • 我个人使用的是windows x64，之后的都是基于这个系统下的操作
  • 首先你得有个github账号，然后还要创建个一个仓库。注意仓库名和账号名需要有个对应，举个粟子：我的账号名starriness，我创建的仓库名就是starriness.github.io
  • 接着你得有个域名，域名解析指向你的仓库名
  • 然后你得安装个node环境和下载个git，这里顺便给出官网下载地址 node, git
  • 注意除非有特别说明，否则所有的东西最好还是使用最新的版本，不然会出现一些版本问题
  • 大部分情况下使用默认安装就可以了，有个别疑问的话可以谷歌一下。在安装Git的时候要注意下把添加鼠标右键命令的选项勾上
  • 安装完成后可以在cmd中分别输入 git –version && node –version && npm –version，出现如图所示就表示已经完全安装成功了

Hexo

hexo是什么呢。。简单来说hexo就是一个可以快速生成博客的框架，它使用markdown来解析文章。就是说你用markdown写完文章后，输入几个指令就能很方便的把你的文章生成为一个静态的网站并部署到github page上

官网：https://hexo.io/
中文官网：https://hexo.io/zh-cn/
GitHub: https://github.com/hexojs/hexo

几个常用的命令: 

hexo clean   //用于清除缓存 
hexo g   //用于生成
hexo d   //用于部署到github
hexo s   //启动本地服务 localhost:4000
hexo new [layout] <title>    //生成一篇新的文章，关于layout的说明参考官网，粟子：hexo new post test

安装hexo

建立一个文件夹后，打开gitbash，在命令行中分别输入 

npm install hexo-cli -g
hexo init
npm install

安装完成后可以输入在gitbash中输入hexo -v 如安装成功则可以看到有输出hexo的版本信息，同时路径下会生成hexo的目录树

部署前准备

  • 为了方便使用git把本地的生成的静态网站部署到GitHub上，需要生成一下本地的ssh密钥
  • 在GitBash中，输入 ssh-keygen -t rsa -C "你的邮箱地址", 连续三次回车。(这样是空密码，当然如果你想自己设置密码的话就在第一次回车后输入密码回车并再次输入密码再回车)
  • 如上图所示即是成功生成了密钥
  • 然后进入 C:\Users\你的用户名.ssh 下面，找到 id_rsa.pub ，用文本编辑器(强烈安利sublime text)打开,复制所有内容
  • 打开 https://github.com/settings/ssh, 把上面复制的内容粘贴到github里去

  • 在gitbash中输入 ssh -T git@github.com 如下图所示则是设置成功

关于部署之后需要重新设置域名

在source目录下创建文件CNAME(没有后缀名),输入你的域名并保存(不带http,https,www)

之后的部署则不需要再重新设置域名了
如果域名解析中没有把@指向github page仓库的话，直接在浏览器中输入域名可能会显示网络连接失败

选择主题

打开站点配置文件 _config.yml ，找到theme，并将值改为对应的主题(在themes文件夹下的文件夹名字)

# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: next

Next

添加标签页和分类页

  • 分别输入指令hexo new page tags和 hexo new page categories
  • 在source目录下可以到看已经生成了对应的文件夹目录
  • 进入目录下的index.md进行编辑(以标签为例，分类同理)
---
title: 标签
date: 2018-02-25 09:46:13
type: "tags"
comments: false
---
  • 在编写博客文章的时候需要在文件最上方的—区域上添加上tags和categories的值 
---
title: 使用GitHub Page+Hexo搭建个人博客
date: 2018-02-24 17:49:47
tags: 
    - hexo
    - GitHub Page
    - NexT
    - 博客
categories: 
    - hexo
description: 记录一下github page的使用和一些踩过的坑
---
