也说说原型和继承

发表于 2018-03-05 | 分类于 前端
毕竟原型，继承这些，算是js中的底层的基础，也说说自己的理解吧

创建对象的方式

Javascript中，创建对象有好几种方式，这里分别说一下

// 使用object函数创建
1. var a = new Object();  
// 字面量表示法
2. var b = {name: "b"};
// 构造函数
3. var C = function(){this.name = 'c'}
   var c = new C();
// 使用Object的create方法
4. var d = Object.create({name: 'd'})

那么，上面的第1和2两是基本是同一类的创建方法，他们的不太好的地方就是创建的对象只适合临时使用，并不能作为模板类来进行调用。
第三类是先创建一个构造函数，再通过new 一个构造函数来创建一个对象，这时此构造函数就形成了新建对象的模板类。可以通过多个new 来创建出多个对象，这里字面量创建法所不能的。
第四类是使用指定的原型对象及其属性去创建一个新的对象。

原型对象

原型规则

  1. JS中万物都是对象，都具有对象的特性，可自由扩展属性(null 除外)
  2. 所有的引用类型(arr, obj, fn)都具有一个 __proto__ 属性，其值为一个普通对象
  3. 所有的函数，都具有一个 prototype 属性，其值为一个普通对象
  4. 所有的引用类型，其 __proto__ 属性值默认指向它的构造函数的 prototype 的属性值

什么是原型对象

原型对象有两种，分别为显式原型prototype 以及隐式原型__proto__。其指向自身所有的共享属性和方法。 原型对象的作用，就是定义所有对象实例所共享的属性和方法。

显式原型

所谓显式原型对象，就是指当我们创建一个函数的时候，JS自身默认会为该函数创建的一个名字为prototype的对象, prototpye里含有一个constructor的键，其值默认指向函数本身。

隐式原型

在js中，万物皆对象，而只要是对象，都会含有一个__proto__的属性，指向创建这个对象的构造函数的原型prototype

var Test1 = function(){ console.log('Test1') };
Test1.prototype.constructor === Test1;  // true
var test1 = new Test1();
test1.__proto__ === Test1.prototype;    // true
// 在js中万物皆对象，数组也是对象，指向的是Array这个构造函数的原型对象
var array1 = [1,2,3,4,5,6,7];
array1.__proto__ === Array.prototype
// 同理，object指向的就是Object这个构造函数的原型对象
var obj1 = {name: 'obj'}
obj1.__proto__ === Object.prototype

JS读取对象时的机制

当JS读取某对象的属性时，会先自对象实例本身开始进行搜索，查询到目标属性，则返回属性的值。如果在实例本身没有查询到，则继续搜索对象指向的原型对象__proto__，以此循环，一直查找到Objec.prototype, 如果这里都没有查找到的话，则会返回undefined。
这里存在一个优先级和层级的问题，由于JS的搜索查询是在实例对象本身开始的，再进行到原型对象，故当存在有实例对象与原型对象同样的属性名的时候，会以实例对象优先

var Test1 = function(){ this.name = 'first name'}
Test1.prototype.name = 'last name';
Test1.prototype.age = 27;
var test1 = new Test1();
console.log(test1.name);    // first name
console.log(test1.age);    // 27
test1.age = 30;
console.log(test1.age);    // 30
delete test1.age;
console.log(test1.age);    // 27

如上面代码所示，test1为Test1的实例对象，Test1为test1的构造函数。
调用name这个属性的时候，先开始在test1实例自身搜索，发现存在有name这个属性，返回其值为first name。
调用age的时候，在实例自身搜索不到，则转向期原型对象中搜索，并返回值为27。
接下来我们往test1实例用添加了一个属性age值为30，再次调用的时候由于实例自身有这个属性，所以返回了30。
再然后我们delete掉了这个属性，再次调用的时候，发现实例自身没有，故向原型对象中搜索，返回值为27

new的工作原理

当使用new来创建一个实例对象的时候，分别经过下面的步骤:

  1. 创建一个新的对象，新对象继承构造函数的原型对象
  2. 构造函数将被执行，同时this上下文被修改，指向实例对象
  3. 返回实例对象。如果构造函数中存在return 一个对象的话，则这个对象会取代整个new的返回结果
var Test1 = function(){ 
    var o = {name: 'test1'}; 
    this.age = 17; 
    var j = {name: 'j1'}; 
}
var test1 = new Test1();
console.log(test1.name);    // undefined
console.log(test1.age);    // 17

var Test1 = function(){ 
    var o = {name: 'test1'}; 
    this.age = 17; 
    var j = {name: 'j1'}; 
    return o;
}
var test1 = new Test1();
console.log(test1.name);    // test1
console.log(test1.age);    // undefined

原型链

如上面所说，每个构造函数都含有一个原型对象prototype，原型对象中都包含着一个指向构造函数本身的指针，而实例都包含着一个指向原型对象的内部指针__proto__。那么，如果让原型对象指向另外一个实例，那么，此时，原型对像将包含一个指向另外一个原型的指针。相应的，另外一个原型中也包含着一个指向另外一个构造函数的指针。如此类推，则构成了原型的链条，即所谓的原型链
粟子：

function Boy(){
    this.name = 'Suven';
    this.status = 'boy';
}
Boy.prototype.school = 'GDPZ';
Boy.prototype.age = '17';
Boy.prototype.buff = function(){
    return ('younger');
}

function Man(){
    this.status = 'man';
    this.say = function(){
        return ('grow up');
    }
}
Man.prototype.age = '27';
Man.prototype.buff = function(){
    return ('older');
}

// 这里实现继承
Boy.prototype = new Man();
var b = new Boy();
console.log(b.name)     ==> 'Suven'
console.log(b.school)   ==> undefined   // Man()的prototype中没有school属性
console.log(b.age)      ==> '27' //这里的age被Man构造函数中的同名属性覆盖
console.log(b.status)   ==> 'boy' //原理同上
console.log(b.say())    ==> 'grow up'  
conosle.log(b.buff())   ==> 'older'

构造函数Boy()中并没有say, Boy.prototpye = new Man()后，Boy.prototype 指向 Man()的实例对象，所以我们也可以调用到say()方法，上面我们说到__proto__指向创建这个对象的构造函数的原型，故Man()的实例对象的__proto__指向它本身的prototype，所以，这里的构造函数Boy()的prototype被重新指向后，它原来的prototype将不再被调用，故b.school输出undefined，JS的调用机制我们上面有说到，这里b.status在Boy()中被检索到并返回，故b.status输出为boy

JS的继承

js的继承方式基本上，有6种，分别说说它们的优点和缺点

借助构造函数继承

function P(){
    this.name = 'p';
}

function C(){
    P.call(this);
    this.type = 'c';
}

var c = new C();
console.log(c.name)  // p
console.log(c.type)  // c

call()方法的作用在于，改变函数的this指向，构造函数C 中的call改变了 构造函数P的this指向，指向了 构造函数C，使其继承了P的属性和方法。apply()的作用同理。

缺点

父类原型对象上的属性和方法无法被继承，并且方法都在构造函数中被定义，因此函数无法复用。

借助原型链继承

function P(){
    this.name = 'p';
}
P.prototype.say = function(){ return 'say p' }

function C(){
    this.type = 'c';
}

C.prototype = new P();

var c = new C();
console.log(c.name)     // p
console.log(c.type)     // c
console.log(c.say())    // say p

缺点

由于C的原型对象是指向了P的proto， 所以C实例化后，对引用类型的操作实际上是对P的内存的值的修改，所以包含引用类型(arr, obj, fn)的原型属性会被所有的实例共享

function P(){
    this.name = 'p';
    this.list = [1,2,3]
}
P.prototype.say = function(){ return 'say p' }

function C(){
    this.type = 'c';
}

C.prototype = new P();

var c1 = new C();
c1.list.push(4);
console.log(c1.list);   // [1,2,3,4]
var c2 = new C();
console.log(c2.list);   // [1,2,3,4]

组合方式继承

这种是实例面向对象继承的最常见方式。

function P(){
    this.name = 'p';
    this.list = [1,2,3]
}
P.prototype.say = function(){ return 'say p' }

function C(){
    P.call(this)
    this.type = 'c';
}

C.prototype = new P();
C.prototype.constructor = C; //如果不指向回C，会被指向到父类

var c1 = new C();
c1.list.push(4);
console.log(c1.list);   // [1,2,3,4]
var c2 = new C();
console.log(c2.list);   // [1,2,3]

缺点

父类的构造函数执行过多。上例中，call()方法的时候被执行，子类 实例化的时候，在子类原型链上被执行

组合方式继承-进阶

function P(){
    this.name = 'p';
    this.list = [1,2,3]
}
P.prototype.say = function(){ return 'say p' }

function C(){
    P.call(this)
    this.type = 'c';
}

C.prototype = P.prototype;
C.prototype.constructor = C; //如果不指向回C，会被指向到父类

var c1 = new C();
c1.list.push(4);
console.log(c1.list);   // [1,2,3,4]
var c2 = new C();
console.log(c2.list);   // [1,2,3]

组合方式继承-进阶

function P(){
    this.name = 'p';
    this.list = [1,2,3]
}
P.prototype.say = function(){ return 'say p' }

function C(){
    P.call(this)
    this.type = 'c';
}

C.prototype = Object.create(P.prototype);
C.prototype.constructor = C; 

var c1 = new C();
c1.list.push(4);
console.log(c1.list);   // [1,2,3,4]
var c2 = new C();
console.log(c2.list);   // [1,2,3]
